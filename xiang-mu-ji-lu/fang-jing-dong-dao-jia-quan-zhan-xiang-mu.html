    # 写在开始

    • 这个项目写了大概2个月，断断续续的写，到最后整个下单流程思路还是比较清晰，这是一个基于vue3.x全家桶（包含最新的vuex和vue-router）的项目，令我最惊喜的莫过于组合式API，它就像是一个逻辑的中转站，这让整个组件的逻辑很清晰，后期维护修改的难度大大减小，当然现在这只是一个纯前端项目，后期会利用node.js赋能，令我感到惊讶的是前端的逻辑也复杂的离谱。
    • 代码已开源，放在码云上，有兴趣的可以扒下来看下：

    https://gitee.com/zhangfanhang/jingdong-vue3.x.git

    # 前端部分

    可以利用vue-cli方便的创建vue项目，项目创建好后安装 normalize.css，它让不同的浏览器在渲染网页元素的时候形式更统一

    另外还要设置好rem适配，rem是一个相对于html的单位，1rem = html下font-size的大小

    // base.scss
html{
   font-size:100px
}
body{
   font-size: 13px;
}

    # flex + iconfont 完成首页 docker 样式编写

    首先不得不提一下阿里巴巴的iconfont，它可以让开发者轻松解决小图标问题。当然这个文件需要在main.js全局引入。现在就利用flex布局完成底部docker的开发

    // iconfont.css
@font-face {
  font-family: 'iconfont';  /* Project id 2494335 */
  src: url('//at.alicdn.com/t/font_2494335_2r15a9q6c36.woff2?t=1626822481533') format('woff2'),
       url('//at.alicdn.com/t/font_2494335_2r15a9q6c36.woff?t=1626822481533') format('woff'),
       url('//at.alicdn.com/t/font_2494335_2r15a9q6c36.ttf?t=1626822481533') format('truetype'),

}
  .iconfont {
    font-family: "iconfont" !important;
    font-size: .2rem;
    font-style: normal;
    -webkit-font-smoothing: antialiased;
    -moz-osx-font-smoothing: grayscale;
  }

    直接一手docker绝对定位，flex。然后四个部分直接flex:1,舒服了，浮动,狗都不用。

    scss是一个css的预编译器，我们可以通过更少的代码实现css代码 通过绝对定位我们很轻松的把docker排除在外， overflow-y: scroll;可以让我们在竖直方向滚动整个页面

    .wrapper {
  // 纵向滚动
  overflow-y: scroll;
  position: absolute;
  bottom: 0.5rem;
  left: 0;
  top: 0;
  right: 0;
  padding: 0 0.18rem;
}

    整个首页的布局，实现起来还是比较简单的。另外还要注意对每个组件的css作用域进行约束

    <style lang="scss" scoped>

    最后对代码进行拆分，也就是组件化，比如附近店铺下的商铺信息就可以作为一个组件，在商铺详情中复用。

    # 登录&注册页开发

    登录和注册页的页面就不多说了，这里主要说一下逻辑部分，这里就要用到vue-router中的路由守卫了，这个还是比较难理解的。

    首先我们先看一下登录逻辑的实现。

    const useLogin = (changeToast) => {
  const router = useRouter()
  const data = reactive({ username: '', password: '' })
  const handleLogin = async () => {
    try {
      const { username, password } = data
      if (username === '') {
        changeToast('请输入用户名')
        return
      }
      if (password === '') {
        changeToast('请输入密码')
        return
      }
      // await 表达式会暂停当前 async function 的执行，等待 Promise 处理完成。若 Promise 正常处理(fulfilled)，其回调的resolve函数参数作为 await 表达式的值，继续执行 async function。
      const result = await post(
        '/api/user/login',
        {
          username: username,
          password: password
        }
      )
      if (result?.errno === 0) {
        localStorage.isLogin = true
        router.push({ name: 'Home' })
      } else {
        changeToast('登录失败')
      }
    } catch (e) {
      changeToast('请求失败')
    }
  }
  const { username, password } = toRefs(data)
  return { username, password, handleLogin }
}

    登录逻辑中，使用了async和await，它的作用就是说可以让异步代码，看起来像同步代码，其中比较难理解的就是await 表达式会暂停当前 async function 的执行，等待 Promise 处理完成。若 Promise 正常处理(fulfilled)，其回调的resolve函数参数作为 await 表达式的值，继续执行 async function。这个点当时困扰了我很久，查阅了MDN文档才恍然大悟。

    回到最重要的部分，路由。

    // 全局前置守卫
router.beforeEach((to, from, next) => {
  const { isLogin } = localStorage;
  // 判断用户是否登陆
  (isLogin || to.name === 'Login' || to.name === 'Register') ? next() : next({ name: 'Login' })
})

    三个条件满足其一，路由就会放行.不满足就会跳转到登录页
